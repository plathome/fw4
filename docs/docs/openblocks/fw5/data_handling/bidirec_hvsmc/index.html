双方向対応高圧スマートメータ設定

PD Handler HVSMC(双方向対応)を用いて双方向対応高圧スマート電力量メータ(Bルート)からECHONET LiteとAIFの仕様に基きデータを取得し、PD Repeater を介してクラウドに送ります。
IoTデータ⇒送受信設定タブの双方向対応高圧スマートメータ(Bルート)設定ペインから設定を行うことが出来ます。
双方向対応高圧スマートメータ(Bルート)設定ペインが表示されていない場合は、モジュール設定を参照し、PD Handler HVSMC(双方向対応)を使用するに設定して下さい。
PD Handler HVSMC(双方向対応) は双方向対応高圧スマート電力量メーターからネットワーク経由で通信する場合には、UDPポート(3610)を用いてデータを待ち受けます。そのため、同ポート開放しておく必要があります。通常のネットワーク以外(特定小電力モジュール等)の場合には、この限りではありません。
システム⇒フィルタータブのフィルター開放設定ペインにおいてECHONET Liteを有効に設定して下さい。
また、ネットワーク経由の場合には使用するインターフェースはネットワークタブにて有効にしてください。

caution

ECHONET Lite認証及びAIF認証が取得できているハードウェアはOpenBlocks IoT VX2、OpenBlocks IoT FX1、OpenBlocks IoT FX1/Eとなります。
それ以外のハードウェアでは動作はしますが、認証はとれておりません。そのため、正式運用時についてはご相談ください。

データの取得タイミング#

PD Handler HVSMC(双方向対応)は次の4つのタイミングでデータを取得し、PD Repeaterに送ります。

  1. PD Handler HVSMC(双方向対応)の起動時
  2. 双方向対応高圧スマート電力量メータの自律動作による30分毎定時計測
  3. PD Handler HVSMC(双方向対応)に設定された取得時間間隔に基づく計測値取得
  4. 双方向対応高圧スマート電力量メータからの動作状態通知時

以下に各取得タイミングにおける取得データとシーケンスを示します。
表中、EPCはECHONET Propaty Codeの略称です。
